# BetterBase Auth Template

This template contains the authentication files generated by `bb auth setup`. It serves as a reference for understanding what the CLI creates and can be used as a fallback or documentation.

## Overview

The `bb auth setup` command generates a complete authentication system with:

- **Session-based authentication** with secure password hashing
- **User signup, login, and logout** endpoints
- **Auth middleware** for protecting routes
- **Database schema** for sessions and accounts tables

## Files

### `src/routes/auth.ts`

Hono route handler for authentication endpoints:

- `POST /auth/signup` - Register a new user with email and password
- `POST /auth/login` - Authenticate user and create session
- `POST /auth/logout` - Invalidate session token

### `src/middleware/auth.ts`

Authentication middleware functions:

- `requireAuth()` - Requires valid session token, returns 401 if missing/invalid
- `optionalAuth()` - Attaches user to context if token present, but doesn't require it
- `getUser(c)` - Helper to extract user from context

### `src/db/auth-schema.ts`

Drizzle ORM schema additions for authentication:

- `sessions` table - Stores active user sessions with expiration
- `accounts` table - Stores OAuth provider connections (for future use)

### `src/auth/index.ts`

BetterAuth server setup file (reserved for future BetterAuth integration).

## Usage

After running `bb auth setup`:

1. Set `AUTH_SECRET` in your `.env` file
2. Run `bun run db:push` to apply schema changes
3. Use `requireAuth` middleware on protected routes:

```typescript
import { requireAuth, getUser } from './middleware/auth';

app.use('/api/protected/*', requireAuth);

app.get('/api/protected/profile', (c) => {
  const user = getUser(c);
  return c.json({ user });
});
```

## Session Management

- Sessions expire after 30 days by default
- Session tokens are passed via `Authorization: Bearer <token>` header
- Passwords are hashed using Bun's built-in `Bun.password.hash()` and `Bun.password.verify()`

## Notes

- This template matches exactly what `bb auth setup` generates
- Do not modify these files directly - they are overwritten on each `bb auth setup` run
- For custom auth logic, create separate files and import from these
